# BeeCount 云协作封存状态（2026-02）

## 目标
- 当前版本以“实验态可用 + 可回退 + 可排障”为基线封存。
- 成员管理继续后台直管，App 保留轻量协作入口。

## 已完成
- BeeCount Cloud 协作读写主链路已接入（read/write/sync/attachments/devices）。
- 共享账本成员查看、设备会话、头像跨端读取已可用。
- 权限模型按 `Owner / Editor / Viewer(历史兼容)` 执行。
- 共享账本写入失败、scope 缺失、映射冲突具备可见提示。
- 账本 identity 自动修复可处理常见 `ledger_*.json` 冲突场景。

## 本次封存收口
- 停用按 `name + currency` 自动合并共享账本，防止同名误吞并。
- 个人账本与共享账本 identity 解耦，优先按远端 `ledger_id(syncId)` 对齐。
- 协作功能维持实验态：入口默认可见，写入前增加统一 Beta 提示。

## 已知限制
- 历史脏数据（同名多候选）仍可能导致部分账本需人工核对。
- App 端权限判断以本地缓存和远端解析混合，偶发短时滞后仍可能出现。
- 协作 E2E 自动化尚未覆盖完整矩阵，当前以手工验收为主。

## 排障建议
1. 先重登 BeeCount Cloud 刷新 token 与用户态。
2. 在云同步页手动执行一次上传或下载。
3. 协作页复制诊断信息（ledgerId/role/scope/deviceId）用于定位。
4. 若仍异常，优先核对本地 `sync_id` 与远端 `ledger_id` 是否一一对应。

## 重启路线（后续）
1. 收敛权限判断到单一数据源（优先远端角色）。
2. 追加协作冲突与映射修复自动化测试。
3. 分阶段将实验提示升级为稳定版门禁（按错误率灰度）。
