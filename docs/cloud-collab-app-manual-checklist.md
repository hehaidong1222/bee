## 前置条件
- BeeCount Cloud 已启用并可登录。
- 至少准备 2 个可登录账号（Owner + 非 Owner）。
- 至少准备 2 台设备用于会话下线验证。
- 至少一个共享账本存在历史 `viewer` 成员（用于兼容验收）。

## 用例清单（必须全通过）
- [ ] 1. Owner 在“账本管理页”可进入共享账本管理入口，仅看到“当前角色 + 成员列表 + 后台管理提示”。
- [ ] 2. App 内不再出现“邀请码管理 / 通过邀请码加入 / 退出共享账本”入口。
- [ ] 3. 成员页在 Owner/Editor/Viewer 下均为只读，不显示改角色或升级按钮。
- [ ] 4. 历史 `viewer` 角色保持只读兼容态（可见不可写）。
- [ ] 5. Web 后台可按邮箱添加成员，App 成员页可看到新增成员。
- [ ] 6. Web 后台可按邮箱更新成员角色，App 角色展示与写权限链路一致。
- [ ] 7. Web 后台可按邮箱移除非 Owner 成员，被移除成员在 App 不再拥有账本访问权限。
- [ ] 8. 云同步页仅保留设备会话入口；设备页远程下线后被下线设备会话失效。
- [ ] 8.1 设备页默认展示“近 30 天去重设备”，切换“全部会话”后数量增加且字段顺序一致（平台/版本/型号/最近活跃）。
- [ ] 9. 两设备并发编辑后最终一致，无重复/丢失。
- [ ] 10. 交易附件与分类自定义图标在另一设备可补齐下载并显示。
- [ ] 11. Mine 页和账本卡可看到共享与角色标识（Android/iOS/桌面一致）。
- [ ] 12. 切换到非 BeeCount Cloud 模式后，共享协作入口不展示且无报错。
- [ ] 13. 当服务端未开启 `ALLOW_APP_RW_SCOPES` 时，角色/设备页展示明确配置提示，不显示原始异常文案。
- [ ] 13.1 协作页可复制“协作诊断信息”（ledgerId/role_resolve_status/scope_hint/deviceId）用于排障。
- [ ] 14. iCloud/WebDAV/S3/Supabase 备份模式下上传/下载流程可完成，且不出现协作入口或协作报错。
- [ ] 15. 共享账本卡显示成员头像堆叠（最多 3 个）与成员数；个人账本不显示成员头像区。
- [ ] 16. 共享账本交易列表每条交易仅显示“创建者头像”（不显示姓名/邮箱）；个人账本不显示创建者信息。
- [ ] 17. 编辑共享账本交易时不显示“创建者信息行”；权限逻辑无回归（Owner 可改全部，Editor 仅可改自己创建）。
- [ ] 18. 共享账本本地新建交易后，列表立即显示当前用户头像，不出现“未知成员创建”。
- [ ] 19. BeeCount Cloud 下上传入口始终可手动点击；若本地队列失败，上传项显示失败计数并可点击重试。
- [ ] 20. App Mine 修改头像后可同步到 Web 成员列表与个人资料；上传失败时云同步页可重试。
- [ ] 21. 账本列表本地账本标题后显示 `ID:<localId>`，远程账本不展示负数占位 ID。
- [ ] 22. 本地默认账本 `id=1` 与远端共享 `ledger_1.json` 冲突时，启动后自动完成映射修复，个人账本不被误判为共享。
- [ ] 23. Web 交易筛选按“用户 + 账本”记忆，刷新/重开后可恢复 `关键词/类型/账户` 条件且互不串扰。
- [ ] 24. 共享账本身份映射不再使用 `name + currency` 自动合并，避免同名账本被吞并。
- [ ] 25. 协作入口页显示“实验功能”提示；共享写入前出现统一实验提示 toast。

## 上线已知限制（可接受）
- App 不提供成员增删改与邀请码流程，成员运维统一在 Web 后台完成。
- App 仅保留历史 `viewer` 兼容读取，不提供新建或降级入口。
- 暂不提供完整协作 E2E 自动化矩阵，当前以手工验收为主。
- 历史 `viewer` 仅做兼容保留，新建流程不推荐继续使用。

## 回滚策略
- 功能回滚：关闭协作入口 Flag 或回退到上一稳定版本。
- 数据回滚：按版本指引恢复本地备份（如已生成迁移前备份）。
- 应急止损：先禁用协作入口，再进行数据核对与修复。

## 用户自助排障步骤
- 1. 退出并重新登录 BeeCount Cloud。
- 2. 在云同步页执行一次手动上传或下载（会触发媒体补齐重试）。
- 3. 若提示权限未就绪，检查服务端是否设置 `ALLOW_APP_RW_SCOPES=true`，重启后重新登录。
- 4. 检查网络与账号权限（是否为 Owner/Editor）。
- 5. 若仍失败，导出日志并附上账本 ID、设备 ID、失败时间点。
